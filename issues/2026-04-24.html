TypeScript 7.0 Beta — 编译器迁移至 Go，构建速度约 10 倍提升

🔥 头条速递

今天最大的新闻是 TypeScript 发了 7.0 Beta —— 但标题不够劲爆，内核才是重点：编译器从 TypeScript 自举改用 Go 重写了。

微软给出的数据是构建速度相比 6.0 提升约 10 倍。新的 tsgo 二进制和 @typescript/native-preview npm 包已经可以试用。类型检查语义保持不变，现有测试套件全量通过，所以你的代码不需要改。

这不只是一个性能优化 —— 这是一个信号：当 TypeScript 团队选择 Go 而不是继续自举，说明他们认为编译器的瓶颈已经不是「类型系统设计」而是「工程效率」。10 倍的提速对大型 monorepo 来说不是锦上添花，而是改变工作流的级别。

与此同时，Epoch AI 和 METR 发布的 MirrorCode 基准测试验证了另一个方向：AI 已经能完成一些需要数周的大型编码任务了。 Claude Opus 4.6 在测试中成功从零重新实现了一个约 17000 行的 Go 生物信息学工具包，仅凭执行访问和测试用例，不看源码。

判断： 编译器在变快，AI 在变强。两者叠加意味着「从想法到可运行代码」的路径正在急剧缩短。开发者的角色正在从"写代码"向"设计系统 + 审查输出"迁移。

📰 科技动态

1、TypeScript 7.0 Beta：编译器用 Go 重写，构建提速 10 倍

微软发布 TypeScript 7.0 Beta，底层编译器从 TypeScript 自举迁移到 Go 实现，激进并行化带来约 10 倍构建速度提升。新的 tsgo 二进制可独立运行，@typescript/native-preview 包可直接 npm 安装体验。类型检查器语义与现有测试套件完全对齐，不需要修改现有代码。这是 TypeScript 项目历史上最大的一次底层架构变更，将深刻影响编辑器体验、CI 流水线和大型 monorepo 的日常开发。

2、Vercel 四月安全事件：OAuth 集成被利用，部分客户环境变量泄露

Vercel 发布安全公告，一起始于第三方 OAuth 集成恶意软件的攻击最终导致部分客户的环境变量被暴露。公告详细走读了攻击链，并强调了 SaaS 工具链中 OAuth 授权粒度和供应链安全的重要性。如果你的项目用 Vercel 部署并授权了第三方 OAuth 应用，建议立即检查并轮换密钥。

3、Node.js 默认启用 Temporal API

Node.js 上游 PR 正在跟进将 ECMAScript Temporal API 默认启用，对齐 Node 26 的时间线。Temporal 是 JavaScript 等了十年的现代日期/时间处理方案，终于不用再 new Date() 然后祈祷时区不出错了。这个变更将影响整个 JS 生态的日期处理习惯。

4、15000 个循环依赖是怎么解的

一位微软工程师记录了在一个 700 万行 TypeScript monorepo 中解开约 15000 个项目级循环依赖的全过程。使用 Nx 的依赖图分析工具系统性地识别和消除循环，分享的方法论适用于任何面临构建顺序和架构腐化问题的大型项目。

5、UK Biobank 数据泄露镜像：基因组数据不该放在 GitHub

一个独立项目揭示了敏感的 UK Biobank 风格健康数据集不断出现在公开 GitHub 仓库中的问题，并搭建了可搜索的镜像供研究者和审计人员使用。"只是在 repo 里放个 CSV" 在涉及人类基因组数据时，可能带来巨大的伦理和法律后果。
